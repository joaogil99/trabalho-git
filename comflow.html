O primeiro passo é baixar e instalar o Git, que está disponível em:
https://git-scm.com/downloads

Para executar uma inicialização rápida do Gitflow e trabalhar com alguns branches é necessário o uso do comando em sequência:
git flow init

As features são branches criados à partir do develop que contém as novas features. Eles serão integrados ao branch develop (que é criada na inicializaçã).

Para iniciar uma nova feature:
git flow feature start MYFEATURE

Para finalizar a feature que está em desenvolvimento:
git flow feature finish MYFEATURE

Para publicar a feature atual com os demais colaboradores:
git flow feature publish MYFEATURE

Para realizar um pull e atualizar uma feature publicada no repositório remoto:
git flow feature pull MYFEATURE

As releases são as branches que possuem o papel de publicar novas versões secundárias ou principais, mas não patches. São mais confiáveis que a branche develop, pois, estão sendo preparadas para integração com a main.
Outro papel importante da release é a definição de versionamento (as tags).

Para criar uma nova release:
git flow release start RELEASE [BASE]
[BASE] é utilizado caso opte ao uso de uma hash sha-1.

O mais indicado após a criação de uma nova release é a publicação do projeto, pois, permite a interação de outros desenvolvedores.
Para publicar a nova release:
git flow release publish RELEASE

Para finalizar a release atual:
git flow release finish RELEASE

O Hotfix é utilizado para corrigir bugs na versão estável mais recente do projeto, não importa se foi um patch principal, secundário ou outro.
É importante lembrar que você pode ter apenas um hotfix ao mesmo tempo.

Para iniciar um novo hotfix:
git flow hotfix start VERSION [BASENAME]
O argumento version marca o nome do novo hotfix. Opcionalmente, você pode especificar um basename para começar.

Para finalizar um hotfix:
git flow hotfix finish VERSION
